# AmorWorks Showcase

AmorWorksとして制作したホームページ、問い合わせ導線、予約導線、運用サポートのビルド事例を紹介する静的サイトです。

公開URL: https://amorworks.github.io/showcase/

## 構成

- `index.html`: 公開ページ本体
- `styles.css`: デザイン
- `script.js`: モバイルメニュー、事例フィルター、年表示、相談メモコピー
- `assets/`: ロゴと事例ビジュアル
- `.nojekyll`: GitHub Pages用

## よく更新する場所

- 事例カードを追加・修正する: `index.html` の `#works` セクション
- 制作プランや金額を変える: `index.html` の `#plans` セクション
- 制作の流れを変える: `index.html` の `#flow` セクション
- 相談メモの内容を変える: `index.html` の `#contact` セクション
- 色、余白、カードの見た目を変える: `styles.css`
- ボタンやコピー機能の動きを変える: `script.js`
- 画像を差し替える: `assets/` に画像を入れて、`index.html` の画像パスを変更する

## 更新の流れ

1. ローカルで該当ファイルを編集する
2. ブラウザでPC幅とスマホ幅を確認する
3. 空リンク、誤字、画像切れがないか確認する
4. `git status` で変更ファイルを確認する
5. `git add .`、`git commit -m "変更内容"`、`git push` の順で反映する
6. GitHub Pagesの公開URLで反映を確認する

## 公開・更新チェック

- 個人情報、住所、電話番号、メールアドレスを公開してよいか確認する
- 事例リンクが実際に開けるか確認する
- 料金が税込、税別、目安、正式価格のどれなのか確認する
- 問い合わせ先のLINE、メール、Googleフォームが正しいか確認する
- スマホで文字がはみ出していないか確認する
- メール直リンクや空リンクを不用意に入れない。未接続の導線は準備中として扱う

## 公開後に整えること

- 問い合わせ先をLINE、メール、Googleフォームのどれにするか決める
- 追加掲載する事例を決める
- 料金表の最終表現を決める
- SNSプロフィールからこのページへ導線を作る

## 問い合わせ先を接続するとき

現在のContact欄は、相談メモをコピーしてLINEやメールに貼り付ける前提です。
受付先が確定したら、`index.html` の `#contact` セクションに実リンクを追加します。

候補:

- LINE公式またはLINEリンク
- Googleフォーム
- メールアドレス
- SNSのDMリンク

接続する前に、リンク先が正しいアカウントか必ず確認してください。

## GitHub Pages運用

このリポジトリは `main` ブランチの `/root` から GitHub Pages で公開します。
独自ドメインは使わず、GitHub Pagesの標準URLで運用します。

反映に数十秒から数分かかることがあります。push直後に古い表示のままでも、少し待って再読み込みしてください。
