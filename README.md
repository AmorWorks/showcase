# AmorWorks Showcase

AmorWorksとして制作したホームページ、問い合わせ導線、予約導線、運用サポートのビルド事例を紹介する静的サイトです。

公開URL: https://amorworks.github.io/showcase/

## 現在の納品状態

- GitHub Pagesで公開済み
- 独自ドメインなし
- 事例2件の公開リンク接続済み
- ホームページ制作プラン、GAS別プラン、制作の流れ、連絡先・お申し込みフォームを掲載済み
- 公式LINEは準備中。フォームは現在コピー式の仮受付
- 指定アドレスへの自動送信は、GASまたは外部フォームサービス接続後に有効化する

## 構成

- `index.html`: 公開ページ本体
- `styles.css`: デザイン
- `script.js`: モバイルメニュー、事例フィルター、年表示、フォーム内容コピー
- `assets/`: ロゴと事例ビジュアル
- `00-資料/`: 入稿用データとサイトブラッシュアップ資料の保管場所
- `.nojekyll`: GitHub Pages用

## 資料フォルダの使い分け

- `00-資料/01-入稿用データ`: 外部提出・入稿・確定素材として扱うデータ
- `00-資料/02-サイトブラッシュアップ資料`: デザイン参考、料金表案、構成案、掲載検討用の資料
- 公開ページで使う画像だけ `assets/` にコピーし、英数字のファイル名にする

## よく更新する場所

- 事例カードを追加・修正する: `index.html` の `#works` セクション
- ホームページ制作プランや金額を変える: `index.html` の `#plans` セクション
- GAS・業務アプリの説明を変える: `index.html` の `#gas-plan` セクション
- 制作の流れを変える: `index.html` の `#flow` セクション
- 各種連絡先・お申し込みフォームを変える: `index.html` の `#contact` セクション
- 色、余白、カードの見た目を変える: `styles.css`
- ボタンやコピー機能の動きを変える: `script.js`
- 画像を差し替える: `assets/` に画像を入れて、`index.html` の画像パスを変更する

## 更新の流れ

1. ローカルで該当ファイルを編集する
2. ブラウザでPC幅とスマホ幅を確認する
3. 空リンク、誤字、画像切れがないか確認する
4. `git status` で変更ファイルを確認する
5. `git add .`、`git commit -m "変更内容"`、`git push` の順で反映する
6. GitHub Pagesの公開URLで反映を確認する

## 公開・更新チェック

- 個人情報、住所、電話番号、メールアドレスを公開してよいか確認する
- 事例リンクが実際に開けるか確認する
- 料金が税込、税別、目安、正式価格のどれなのか確認する
- 問い合わせ先のLINE、メール、フォーム送信先が正しいか確認する
- スマホで文字がはみ出していないか確認する
- メール直リンクや空リンクを不用意に入れない。未接続の導線は準備中として扱う

## 公開後に整えること

- 公式LINEのURLを接続する
- フォーム送信先のメールアドレスと送信方式を決める
- 追加掲載する事例を決める
- 料金表の最終表現を決める
- SNSプロフィールからこのページへ導線を作る

## 問い合わせ先を接続するとき

現在のContact欄は、公式LINEを準備中表示にし、お申し込みフォームは入力内容をコピーする仮受付にしています。
指定アドレスへ自動送信する場合、GitHub Pagesだけでは送信処理ができないため、GASまたは外部フォームサービスを接続します。

候補:

- LINE公式またはLINEリンク
- GAS Webアプリでフォーム内容を指定アドレスへ送信
- Googleフォーム
- Formspreeなどの外部フォームサービス
- SNSのDMリンク

接続する前に、リンク先が正しいアカウントか必ず確認してください。

GASで接続する場合は、`script.js` が `data-form-endpoint` を見て送信します。
`index.html` のフォームタグに `data-form-endpoint="GASのWebアプリURL"` を追加し、ボタン文言を「送信する」に変更します。

## GitHub Pages運用

このリポジトリは `main` ブランチのリポジトリ直下 `/` から GitHub Pages で公開します。
独自ドメインは使わず、GitHub Pagesの標準URLで運用します。

反映に数十秒から数分かかることがあります。push直後に古い表示のままでも、少し待って再読み込みしてください。
