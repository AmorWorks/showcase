# AmorWorks Showcase

AmorWorksとして制作したホームページ、問い合わせ導線、予約導線、運用サポートのビルド事例を紹介する静的サイトです。
今後このサイトは `SC` と呼びます。SCはショーケースの略で、案件受注営業用のホームページです。

公開URL: https://amorworks.github.io/showcase/

## 現在の納品状態

- GitHub Pagesで公開済み
- 独自ドメインなし
- 事例3件の公開リンク接続済み
- ホームページ制作5プラン、プラン画像、比較表、各プラン詳細、DX・AI成長ステップ、制作の流れ、連絡先・お申し込みフォームを掲載済み
- 公式LINEは準備中
- フォームは `cattery227.amor.alice@gmail.com` 宛てに送信する設定済み
- 初回送信後、FormSubmitから届く確認メールを承認すると受付が有効になる

## 構成

- `index.html`: Topページ。LPとして全体像と問い合わせ導線を掲載
- `works/`: ビルド事例ページ
- `philosophy/`: AmorWorksの理念・制作判断ページ
- `services/`: ホームページ制作、DX・AI、Webアプリ、運用サポートの説明ページ
- `plans/`: ホームページ制作5プランの料金ページ
- `dx-ai/`: DX・AI支援ページ
- `flow/`: 制作の流れページ
- `contact/`: 相談・お申し込みフォームページ
- `styles.css`: デザイン
- `script.js`: モバイルメニュー、事例フィルター、年表示、フォーム内容コピー
- `assets/`: ロゴ、事例ビジュアル、料金プラン画像
- `00-資料/`: 入稿用データとサイトブラッシュアップ資料の保管場所
- `.nojekyll`: GitHub Pages用

## 資料フォルダの使い分け

- `00-資料/01-入稿用データ`: 外部提出・入稿・確定素材として扱うデータ
- `00-資料/02-サイトブラッシュアップ資料`: デザイン参考、料金表案、構成案、掲載検討用の資料
- 公開ページで使う画像だけ `assets/` にコピーし、英数字のファイル名にする

## 更新の流れ

1. ローカルで該当ファイルを編集する
2. ブラウザでPC幅とスマホ幅を確認する
3. 空リンク、誤字、画像切れがないか確認する
4. `git status` で変更ファイルを確認する
5. `git add .`、`git commit -m "変更内容"`、`git push` の順で反映する
6. GitHub Pagesの公開URLで反映を確認する

## 公開・更新チェック

- 個人情報、住所、電話番号、メールアドレスを公開してよいか確認する
- 事例リンクが実際に開けるか確認する
- 料金が税込、税別、目安、正式価格のどれなのか確認する
- 問い合わせ先のLINE、メール、フォーム送信先が正しいか確認する
- スマホで文字がはみ出していないか確認する
- メール直リンクや空リンクを不用意に入れない。未接続の導線は準備中として扱う

## 公開後に整えること

- 公式LINEのURLを接続する
- FormSubmitの初回確認メールを承認する
- 追加掲載する事例を決める
- 料金表の公開表現を最終確認する
- SNSプロフィールからこのページへ導線を作る

## 問い合わせ先を接続するとき

現在のContact欄は、公式LINEを準備中表示にし、お申し込みフォームはFormSubmit経由で `cattery227.amor.alice@gmail.com` に送る設定にしています。
GitHub Pagesだけではメール送信処理ができないため、外部フォーム受け口を使っています。

候補:

- LINE公式またはLINEリンク
- 現在: FormSubmitでフォーム内容を指定アドレスへ送信
- 将来: GAS Webアプリに置き換えてフォーム内容を指定アドレスへ送信
- Googleフォーム
- Formspreeなどの外部フォームサービス
- SNSのDMリンク

接続する前に、リンク先が正しいアカウントか必ず確認してください。

フォーム送信先を変える場合は、`index.html` のフォームタグにある `data-form-endpoint` を変更します。
GASで接続する場合は、`data-form-endpoint="GASのWebアプリURL"` に差し替えます。

## GitHub Pages運用

このリポジトリは `main` ブランチのリポジトリ直下 `/` から GitHub Pages で公開します。
独自ドメインは使わず、GitHub Pagesの標準URLで運用します。

反映に数十秒から数分かかることがあります。push直後に古い表示のままでも、少し待って再読み込みしてください。
